# Отчет по лабораторной работе №5

## Тема: Автоматизация сборки проектов на C с использованием GNU Make

**Цель:** Оптимизация процесса компиляции и управления исполняемыми файлами для большого количества упражнений.

### Описание реализации

Для работы с лабораторными работами был разработан универсальный `Makefile`, реализующий следующие функции:

1.**Автоматическое обнаружение (Wildcards):** Скрипт самостоятельно находит все файлы `.c` в поддиректориях `src/lab_*`, что избавляет от необходимости прописывать каждый новый файл вручную.
2.**Маппинг путей (Patsubst):** Реализована логика трансформации путей из `src/` в `builds/`, что позволяет хранить бинарники отдельно от исходного кода, сохраняя структуру папок.
3.**Шаблонные правила:** Использовано правило `$(BUILD_DIR)/%: $(SRC_DIR)/%.c`, которое автоматически создает необходимые директории (`mkdir -p`) перед компиляцией.
4.**Модульная сборка:** Добавлена возможность собирать только файлы конкретной лабы с помощью фильтрации целей: `make lab_5`.

### Результат выполнения

Система сборки успешно протестирована. Команда `make lab_5` автоматически компилирует все упражнения текущей работы с флагами `-Wall -Wextra -pthread`, гарантируя чистоту и воспроизводимость билда.

**Пример команды сборки:**

```bash
make lab_4
```
