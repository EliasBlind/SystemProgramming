# Отчет по практической работе №1: Системный вызов fork()

## Анализ кода

### Используемые библиотеки

* `<stdio.h>`: стандартная библиотека ввода-вывода (используется для функций `printf` и `fprintf`).
* `<unistd.h>`: содержит определения системных вызовов POSIX, включая `fork()`.
* `<sys/types.h>`: содержит определения типов данных, таких как `pid_t`.

### Функциональность кода

Программа демонстрирует создание нового (дочернего) процесса. После вызова `fork()` выполнение разделяется на два параллельных потока (родитель и потомок). Каждый процесс проверяет значение переменной `pid` и выводит соответствующие сообщения в консоль.

### Ответы на вопросы

1.**Возвращаемое значение fork() (родитель):** При успешном создании возвращается **PID (Process ID) дочернего процесса** (положительное целое число).
2.**Возвращаемое значение fork() (потомок):** Возвращается **0**.
3.**Обработка ошибок:**

* **Назначение `if (pid < 0)`:** Проверка, не завершился ли вызов `fork()` неудачей.
* **Ошибки:** Например, `EAGAIN` (достигнут лимит процессов) или `ENOMEM` (нехватка памяти).
* **Важность:** Без обработки программа может продолжить работу в некорректном состоянии, пытаясь использовать неопределенный PID.
4.**Вывод программы:**
* `Привет, мир!` (от дочернего процесса)
* `Привет, мир!` (от родительского процесса)
* `Я родительский процесс` (от родительского процесса)
* *Примечание: порядок строк от разных процессов может меняться в зависимости от планировщика ОС.*
5.**Причины неполного вывода:**
* Ошибка при выполнении `fork()` (сработает блок `pid < 0`).
* Перенаправление потоков вывода или буферизация, при которой один из процессов завершился аварийно до сброса буфера.
* Принудительное завершение одного из процессов системой.
