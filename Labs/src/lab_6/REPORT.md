# Отчет по практической работе №6

## Часть 1. Изучаем дескрипторы процесса

### Последовательность выполнения ч1

1. Выполнена команда `ls -l /proc/self/fd`.
2. В другом терминале запущена команда `sleep 100`.
3. Определен PID процесса `sleep` с помощью `pgrep sleep`.
4. Просмотрены дескрипторы процесса `sleep` командой `ls -l /proc/<PID>/fd`.

### Ответ на вопрос

**Вопрос:** Что означают дескрипторы 0, 1, 2?
**Ответ:**

* **0 (stdin):** Стандартный поток ввода. По умолчанию связан с клавиатурой.
* **1 (stdout):** Стандартный поток вывода. По умолчанию выводит данные в терминал.
* **2 (stderr):** Стандартный поток ошибок. Используется для вывода диагностических и статусных сообщений, также по умолчанию направлен в терминал.

## Часть 2. Перенаправление потоков

### Последовательность выполнения ч2

1. Выполнена команда `ls -l /etc > list_etc.txt`. Список файлов директории `/etc` сохранен в файл.
2. Выполнена команда `date >> list_etc.txt`. Текущая дата добавлена в конец того же файла.
3. Выполнена команда `ls -l /nonexistent_dir 2> errors.log`. Сообщение об ошибке (так как директории не существует) записано в файл `errors.log`.
4. Выполнена команда `ls -l /etc > all_output.txt 2>&1`. Потоки вывода и ошибок объединены и направлены в один файл.

### Ответы

**Вопрос:** В чем разница между операторами `>` и `>>`?
**Ответ:**

* * Оператор `>` (перезапись) создает новый файл или **полностью перезаписывает** содержимое существующего файла.
* Оператор `>>` (дозапись) **добавляет** новые данные в конец существующего файла, не удаляя старую информацию.
