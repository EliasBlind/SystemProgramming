# Отчет по лабораторной работе №4

## Тема: Динамический анализ программного кода

**Цель работы:** Выявление ошибок при работе с памятью в программах на языке C с использованием инструмента динамического анализа **Valgrind**.

### 1. Ход работы

- Разработан bash-скрипт для автоматизированной проверки скомпилированных файлов.
- Выполнена сборка программ `leak`, `overflow`, `hello` и `abs_example`.
- Проведен анализ каждого модуля на предмет утечек памяти (memory leaks) и ошибок доступа.

### 2. Результаты анализа

В ходе тестирования были получены следующие результаты:

| Файл | Статус | Описание |
| :--- | :--- | :--- |
| **leak** | Ошибка | Обнаружена утечка: `definitely lost: 40 bytes`. Память выделена через `malloc`, но не освобождена. |
| **overflow** | OK | Ошибок переполнения буфера или некорректного доступа не выявлено. |
| **hello** | OK | Программа работает корректно, утечек нет. |
| **abs_example** | OK | Ошибок динамической памяти не обнаружено. |

### 3. Вывод

Инструмент **Valgrind** успешно применен для отладки. В модуле `leak` выявлена критическая ошибка управления памятью. Полученные навыки позволяют минимизировать риск появления трудноуловимых багов (Memory Leaks, Segmentation Fault) в системном ПО.
