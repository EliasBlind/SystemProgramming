# Домашняя работа. Тема 2 (Exec)

## Таблица функций семейства exec

| Название функции | Описание | Синтаксис | Отличия |
| :--- | :--- | :--- | :--- |
| **execl** | Заменяет текущий процесс новым, используя список аргументов. | `execl(path, arg0, arg1, ..., NULL);` | Принимает аргументы как **список**, требует `NULL` в конце и указания **полного пути**. |
| **execv** | Заменяет текущий процесс новым, используя массив (вектор) аргументов. | `execv(path, argv[]);` | Принимает аргументы как **массив** указателей, требует указания **полного пути**. |
| **execle** | Заменяет процесс, используя список аргументов и пользовательское окружение. | `execle(path, arg0, ..., NULL, envp[]);` | Позволяет явно передать массив переменных **окружения** (`envp`), аргументы передаются **списком**. |
| **execve** | Системный вызов, заменяющий процесс с массивом аргументов и окружением. | `execve(path, argv[], envp[]);` | Базовая функция: принимает и **массив** аргументов, и **массив** переменных окружения. |
| **execlp** | Заменяет процесс, используя поиск файла в переменной PATH и список аргументов. | `execlp(file, arg0, ..., NULL);` | Ищет исполняемый файл в системном **PATH** (можно не указывать полный путь), аргументы — **списком**. |
| **execvp** | Заменяет процесс, используя поиск в PATH и массив аргументов. | `execvp(file, argv[]);` | Ищет файл в **PATH**, аргументы передаются в виде **массива** (вектора). |

---

## Пояснение к суффиксам

* **l** (*list*) — аргументы передаются списком.
* **v** (*vector*) — аргументы передаются массивом.
* **p** (*path*) — использует переменную PATH для поиска файла.
* **e** (*environment*) — позволяет передать свой массив переменных окружения.
