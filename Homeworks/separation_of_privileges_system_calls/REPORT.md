# Домашняя работа: Разделение привилегий и системные вызовы

## 1. Определения

* **User Mode (Пользовательский режим)** — режим работы процессора с ограниченными правами, в котором выполняются обычные программы. В нем запрещен прямой доступ к оборудованию и критическим областям памяти.
* **Kernel Mode (Режим ядра)** — привилегированный режим работы, в котором у процессора есть полный доступ ко всем ресурсам системы. В этом режиме работает ядро ОС.
* **Системный вызов (syscall)** — специальный интерфейс, позволяющий программе из User Mode запросить услугу у ядра ОС (например, открыть файл или выделить память).
* **Page Cache (Страничный кэш)** — механизм ядра, который хранит данные файлов в оперативной памяти, чтобы ускорить операции чтения и записи, избегая частого обращения к диску.
* **fsync** — системный вызов, который принудительно переносит данные из кэша (памяти) на физический носитель (диск), гарантируя их сохранность.
* **Буферизация** — процесс накопления данных во временной области памяти перед их отправкой или обработкой. Это позволяет объединять множество мелких операций записи в одну большую.

## 2. Правда или ложь

1. **Ложь.** В User Mode программа **не может** напрямую обращаться к устройствам. Для этого она должна использовать системные вызовы.
2. **Ложь.** Системный вызов `write()` обычно пишет данные в кэш (RAM). Гарантированную запись на диск дает только `fsync()`.
3. **Правда.** Инструкция `syscall` переключает состояние процессора из User Mode в Kernel Mode.
4. **Правда.** Выполнение программы приостанавливается до тех пор, пока данные физически не запишутся на диск.
5. **Ложь.** Большинство функций (например, `strlen` или `strcpy`) работают полностью в **User Mode**. В Kernel Mode переходят только те, что запрашивают ресурсы ОС (как `printf` через `write`).
6. **Ложь.** Отсутствие буферизации приводит к огромному количеству системных вызовов, что сильно **замедляет** работу программы из-за постоянных переключений контекста.

## 3. Дополните объяснение

> для создание лог файлов выполните `./code_analize.sh`

1. **Системный вызов — это единственный легальный способ** для программы взаимодействовать с аппаратным обеспечением, файловой системой и другими защищенными ресурсами под контролем ОС.
2. **Если программа вызывает `write()` по одному байту в цикле,** это вызывает критическое падение производительности из-за накладных расходов на миллионы переключений между режимами User Mode и Kernel Mode.
3. **`strace` помогает** отследить, какие системные вызовы делает программа, с какими аргументами и результатами, что незаменимо для отладки и понимания работы приложения «под капотом».

## 4. Практика

### 1. Назначение

* **Программа 1:** Итерационный вывод чисел 0-19 в stdout.
* **Программа 2:** Интерактивный калькулятор (сумма и произведение).

## 2. Системные вызовы

| Вызов | Назначение | Режим |
| :--- | :--- | :--- |
| **execve** | Запуск процесса | Kernel |
| **brk/mmap** | Выделение памяти | Kernel |
| **read** | Ввод данных (в Prog 2 считал `12 12\n`) | Kernel |
| **write** | Вывод текста в консоль | Kernel |
| **lseek** | Попытка навигации по потоку | Kernel |

### 3. Анализ выполнения

* **User Mode:** Логика циклов, вычисление суммы (24) и произведения (144).
* **Kernel Mode:** Все операции ввода-вывода и управление памятью.
* **Ошибки:**
* `access`: `-1 ENOENT` (отсутствие необязательных конфигов).
* `lseek`: `-1 ESPIPE` (в Prog 2). Возникает, так как нельзя «перематывать» ввод из терминала.

## 4. Возвращаемые значения

* `read`: возвращает количество считанных байт (напр. `6`).
* `write`: возвращает количество записанных байт (напр. `51` или `16`).
* `exit_group(0)`: успешное завершение.
